SCP Project

Case Study: Branching Narrative and Dialogue Systems in Twine.

Overview

This project is an original interactive narrative game built in Twine (Harlowe format), set in the SCP Foundation universe. The player takes the role of a Foundation agent navigating encounters with five anomalous entities across episodic chapters, with an underlying macro-narrative about The Scarlett King that emerges through optional classified files.

The game is built entirely in Twine with custom JavaScript for audio (Harlowe Audio Library) and timer mechanics. It is designed as a playable portfolio piece demonstrating branching narrative structure, dialogue system design, variable-based state tracking, and environmental pacing through text.

This page documents the design decisions behind the system, not the story, but the architecture that makes the story work.

Core Systems
  • The Awareness Counter
  • Branching Dialogue Architecture
  • Death States as Narrative
  • Audio as Pacing Tool
What I Have Learned

Each episode has a distinct audio layer implemented through Harlowe Audio. The musical palette was designed before the episodes were written — not to set atmosphere, but to set the player's cognitive state. The Veteran's ambient track is slow and warm. 096's track is minimal. 682's track has urgency built into its tempo. The Scarlett King's theme enters at volume 0 and fades in over thirty seconds. The most significant audio decision was the silence in the 2399 episode after the passive response is sent. Two seconds of silence in a text-based game is an unusual choice. The silence communicates waiting — which is the emotional content of that moment — without a single word.
